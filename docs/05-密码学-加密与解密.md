## 1. 📌 对称加密——用同一把钥匙

### 🏢 生活类比：你家的门锁

```
对称加密 = 你家大门
  锁门和开门用同一把钥匙
  谁有这把钥匙，谁就能进出
  
  问题：怎么安全地把钥匙交给对方？
  你不可能把钥匙通过互联网"递"过去——万一被人截了呢？
```

### AES——现代对称加密的标准

```
AES（高级加密标准）工作流程：

明文: "转账 100 元给小明"
    ↓
    ↓  [AES 加密] + 密钥(16/24/32字节)
    ↓
密文: "xK9#m2!Pq@..."（看起来像乱码）
    ↓ 通过网络传输
    ↓  [AES 解密] + 同样的密钥
    ↓
明文: "转账 100 元给小明"

核心：没有密钥的人，看到密文也解不开
```

### 分组密码的工作模式（ECB vs CBC 图解）

```
ECB模式（电子密码本）——有严重问题！

原始图片:            ECB加密后:
┌────┬────┐         ┌────┬────┐
│ ☀  │ ☀  │         │ @# │ @# │ ← 相同块→相同密文！轮廓泄露！
│ ☀  │ ☁  │         │ @# │ $% │
└────┴────┘         └────┴────┘
（企鹅的轮廓还能看出来！）

CBC模式（密码块链接）——更安全：
每个明文块先用前一个密文块异或一下，再加密。
这样就算两个块内容一样，密文也不同。
```

**网安意义：** ECB 模式是安全面试/考试的高频考点。任何时候都别用 ECB。

## 2. 📌 非对称加密——公钥和私钥

### 🏢 生活类比：街头的邮箱

```
对称加密的问题：你得先把钥匙给对方。钥匙在网络上传送就可能被偷。

非对称加密的解决方案：

  公钥 = 街边邮箱的投递口
    → 谁都可以往里投信（谁都可以用公钥加密）
    → 但只有邮递员有钥匙打开邮箱取信（只有私钥能解密）

  私钥 = 邮递员的钥匙
    → 绝不对外给任何人
    → 只有它能打开用公钥加密的信
```

### RSA——最经典的非对称算法

```
RSA 的核心：两个大质数相乘很容易，但把乘积分解回两个质数很难。

生成密钥：
  1. 选两个大质数 p 和 q
  2. n = p × q          （公钥和私钥都包含 n）
  3. 计算 φ(n) = (p-1)(q-1)
  4. 选 e（公钥指数，常用 65537）
  5. 算 d（私钥指数），使得 e×d ≡ 1 (mod φ(n))

加密：密文 = (明文 ^ e) mod n
解密：明文 = (密文 ^ d) mod n

安全性：知道 n 但不知道 p 和 q，就无法算出 d。
       从 n 分解出 p 和 q 需要天文数字的时间。
```

### 💻 Python 实现 RSA（概念演示版）

```python
# 不要在生产环境用！这只是帮助理解原理的演示代码！
# 生产环境用 cryptography 库

p = 61    # 实际要用几百位的质数
q = 53
n = p * q                              # n = 3233
phi = (p - 1) * (q - 1)               # φ(n) = 3120
e = 17                                 # 公钥指数
d = pow(e, -1, phi)                    # 私钥指数（Python 3.8+）

print(f"公钥: (e={e}, n={n})")
print(f"私钥: (d={d}, n={n})")

# 加密
msg = 123
cipher = pow(msg, e, n)                # 密文 = 123^17 mod 3233
print(f"明文: {msg} → 密文: {cipher}")

# 解密
plain = pow(cipher, d, n)              # 明文 = cipher^d mod 3233
print(f"密文: {cipher} → 明文: {plain}")
```

### 非对称 vs 对称 实际怎么用？

```
HTTPS 的做法（混合加密）：

  1. 用非对称加密(RSA/ECDH) → 安全地在握手阶段传递"会话密钥"
  2. 之后的数据传输 → 用对称加密(AES) + 这个会话密钥
  
  为什么混合？
  非对称加密太慢了！全程用 RSA 加密网页内容会卡死。
  所以只用它传钥匙，传完了换 AES。
```

## 3. 📌 哈希函数——数据的指纹

### 🏢 生活类比：身份证号

```
哈希 = 把任意长度的数据"压缩"成固定长度的摘要

  就像一个超大文件"拍了一张指纹照"：
    → 不同文件 → 不同指纹（理想情况下）
    → 拿到指纹不能反推出原文件（单向）
    → 文件改了1个字节 → 指纹完全变了（雪崩效应）
```

### 五大特性

```
1. 固定长度输出    → 输入可以1字节，也可以1GB，输出长度不变
2. 单向性          → 有哈希值算不出原文（SHA256没有"逆运算"）
3. 雪崩效应        → 输入改1bit，输出面目全非
4. 抗碰撞          → 找不到两个不同输入有相同哈希
5. 快速计算        → 不像 RSA 那样慢
```

### 常见哈希算法

| 算法 | 输出长度 | 安全性 | 现状 |
|------|:--:|:--:|------|
| MD5 | 128bit | ❌ 已破解 | 绝对不要用于安全目的 |
| SHA-1 | 160bit | ❌ 已破解 | 同上 |
| SHA-256 | 256bit | ✅ 安全 | 当前主流 |
| SHA-3 | 可变 | ✅ 安全 | 新一代标准 |

### 💻 试试看

```python
import hashlib

# SHA-256
print(hashlib.sha256(b"hello").hexdigest())
# 输出: 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

print(hashlib.sha256(b"Hello").hexdigest())
# 输出: 185f8db32271fe25f561a6fc938b2e264306ec304eda518007d1764826381969
# 注意：hello → Hello（只改了首字母大写），结果完全不同！这就是雪崩效应。
```

### 🔥 网安中的应用

```
1. 密码存储：数据库不存明文密码，存 hash(password + salt)
   → 即使数据库泄露，黑客也拿不到原始密码

2. 文件完整性校验：下载文件时附带 SHA256 值，你下载后自己算一遍对比
   → 防止文件被篡改

3. 数字签名：先哈希得到摘要，再对摘要签名
   → 签名短小的摘要比签名整个文件高效得多

4. CTF：哈希长度扩展攻击（针对 MD5/SHA1/SHA256 的一种经典攻击）
```

## 4. 数字签名与证书——互联网信任体系

### 🏢 生活类比

```
数字签名 = 你在一份合同上按手印
  → 手印是你独有的（签名用你的私钥生成）
  → 别人能验证这个手印确实是你的（用你的公钥验证）
  → 你不能赖账说不是你按的（不可否认性）

数字证书 = 你的身份证
  → 公安局（CA，证书颁发机构）发的
  → 上面有你的名字和照片（域名和公钥）
  → 有公安局的钢印防伪（CA的签名）
  → 你去办业务，人家看身份证就信你是谁
```

### 签名和验证流程

```
签名（发送方）:
  原文件 → 哈希 → 摘要 → 用私钥加密 → 签名
  发送: 原文件 + 签名

验证（接收方）:
  原文件 → 哈希 → 摘要1
  签名 → 用公钥解密 → 摘要2
  摘要1 == 摘要2？ → 一致 = 未被篡改 + 确实是那个人签的
```

### 证书链——信任的传递

```
根CA证书（预装在操作系统里，无条件信任）
    │
    └→ 签发了 中间CA证书
           │
           └→ 签发了 baidu.com 的证书

浏览器验证 baidu.com 证书时：
  往上追溯到根CA → 根CA是预装的 → 可信！
```

**网安意义：** 如果攻击者能在受害者电脑上安装自己的根证书（或诱骗用户安装），就可以用 BurpSuite 之类工具做 HTTPS 中间人解密。

## 5. 常见密码攻击

| 攻击类型 | 原理 | 生活类比 |
|---------|------|---------|
| 暴力破解 | 试所有可能的密钥 | 0000→0001→...→9999 试密码锁 |
| 字典攻击 | 用常见密码列表试 | 用"常用弱密码TOP1000"挨个试 |
| 彩虹表 | 预计算哈希→密码的巨型对照表 | 把答案抄好，考试直接查 |
| 频率分析 | 密文中出现频率最高的字符可能对应明文高频字符 | 英文里 e 出现最多 |
| 选择明文攻击 | 攻击者可以加密自己选的内容，分析密文规律 | ECB企鹅图就是这个原理 |
| 中间人攻击 | 通信双方中间插一脚，互相对对方冒充 | 假快递员同时骗买家和卖家 |
| 边信道攻击 | 不破算法，利用物理信息（功耗、时间、电磁） | 听按密码锁的声音辨数字 |

### 🔥 密码学的两条黄金法则

> **1. 永远不要自己发明加密算法。**
> 用经过大量专家审计过的标准：AES-GCM、RSA-OAEP、Curve25519。

> **2. 实现时安全≠算法安全。**
> 就算算法是对的，你实现的时候侧信道（时间差异、内存残留）也可能泄露密钥。
