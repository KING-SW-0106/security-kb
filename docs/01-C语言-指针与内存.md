## 1. 内存到底是什么？

### 🏢 生活类比：酒店储物柜

把内存想象成 **一排带编号的储物柜**：

```
柜子编号（地址）:  001   002   003   004   005   006   ...
                 ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
                 │ H │ │ e │ │ l │ │ l │ │ o │ │   │  ← 每个柜子存1个字节
                 └───┘ └───┘ └───┘ └───┘ └───┘ └───┘
```

- 每个柜子能放 **1个字节**（8个比特，0~255）
- 每个柜子有唯一的 **编号**（地址，如 `0x7fff1234`）
- CPU 通过编号来存取数据——就像你拿储物柜钥匙上的号码去找柜子

### 程序中的数据怎么放？

```
char a = 'A';      // 1个字节 → 占1个柜子
int  b = 100;      // 4个字节 → 占4个连续柜子
char str[] = "Hi"; // 3个字节(H,i,\0) → 占3个连续柜子

内存示意：
地址:  0x1000  0x1001  0x1002  0x1003  0x1004  0x1005  0x1006
     ┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐
     │   A   ││  100的 ││  100的 ││  100的 ││  100的 ││  'H'  ││  'i'  │
     │  (a)  ││ 第1字节││ 第2字节││ 第3字节││ 第4字节││ str[0]││ str[1]│
     └───────┘└───────┘└───────┘└───────┘└───────┘└───────┘└───────┘
```

## 2. 指针——存地址的变量

### 🏢 生活类比：快递上的收件人

```
普通变量 a = 10：
    → 你知道"小明有10块钱"（直接知道值）

指针 p = &a：
    → 你知道"小明住在3号楼502"（知道地址）
    → 想找小明？按地址去敲门 → *p（解引用）
```

### 💻 代码示例

```c
int a = 10;         // a 是普通变量，里面存 10
int *p = &a;        // p 是指针变量，里面存 a 的地址

printf("a的值: %d\n", a);     // 输出: 10   （直接拿）
printf("a的地址: %p\n", &a);  // 输出: 0x7fff1234（a放哪了）
printf("p的值: %p\n", p);     // 输出: 0x7fff1234（p里存的就是地址）
printf("p指向的值: %d\n", *p); // 输出: 10   （按地址去拿）
```

### 图解指针

```
        变量 a                        变量 p（指针）
    ┌───────────┐                ┌───────────┐
    │    10     │  ←───────     │ 0x7fff1234 │  ← p 存的是 a 的柜子编号
    └───────────┘                └───────────┘
    地址: 0x7fff1234             地址: 0x7fff1238

    &a → 问"a放几号柜？" → 0x7fff1234
    *p → 问"p存的柜子里有什么？" → 10
```

### ⚠️ 指针为什么是漏洞之源？

```c
int *p;             // 声明了但没赋值！p 里是随机的柜子编号
*p = 100;           // 往那个随机柜子里放 100
// 可能写到了其他程序的地盘 → 崩溃或安全漏洞
```

**这就是缓冲区溢出的根：程序往不该写的地方写了东西。**

## 3. 数组和指针——其实是一回事

### 🏢 生活类比：街道门牌号

```
int arr[5] = {10, 20, 30, 40, 50};

arr → 是"这条街的第一间房的门牌号"（首地址）
arr[0] → 是第一间房（10）
arr[1] → 是第二间房（20）
arr[i] → 从第一间出发，走过 i 间房，那一间的东西
```

### 💻 关键代码

```c
int arr[5] = {10, 20, 30, 40, 50};

// 下面四行完全等价——都是取第3个元素（值为30）
printf("%d\n", arr[2]);     // 正常写法
printf("%d\n", *(arr + 2)); // arr是地址，+2表示往后走2个int
printf("%d\n", 2[arr]);     // 因为 *(arr+2) = *(2+arr) = 2[arr]

// 注意：C语言不检查你是否越界！
printf("%d\n", arr[100]);   // 能编译通过！但读到的是不知道什么东西
```

### 图解数组在内存中

```
int arr[5] = {10, 20, 30, 40, 50};

内存（每个int占4字节）:
地址: arr+0    arr+1    arr+2    arr+3    arr+4
     ┌────────┬────────┬────────┬────────┬────────┐
     │   10   │   20   │   30   │   40   │   50   │
     └────────┴────────┴────────┴────────┴────────┘

arr+0 = &arr[0] = 0x1000
arr+1 = &arr[1] = 0x1004  （+1不是+1字节，是+1个int = +4字节）
arr+2 = &arr[2] = 0x1008
```

**arr[i]** 实际做的是：**从 arr 地址出发，走 i×4 字节，取那里的值。** 如果 i=100，它就走到完全不相关的地方——**C语言不拦你**。

## 4. 📌 栈（Stack）和堆（Heap）——程序的两种内存

### 🏢 生活类比

```
栈(Stack) = 餐厅的盘子摞
  → 后进先出（LIFO），吃一个拿最上面的
  → 自动管理，函数进来就铺新盘子（push），函数走就收走（pop）
  → 大小固定（一般8MB），超出就"栈溢出"了

堆(Heap) = 公共仓库
  → 你要多大、什么时候要，自己申请
  → 用完要自己还（free），不还就是"内存泄漏"
  → 大小受限于物理内存
```

### 图解栈的结构（最关键的一张图！）

```
      高地址（地址数值大）
    ┌────────────────────┐
    │   main 的局部变量    │  ← 调用者的地盘
    ├────────────────────┤
    │   传给 func 的参数    │  ← 函数参数（从右向左压栈）
    ├────────────────────┤
    │   ☆ 返回地址        │  ← func 执行完，该回 main 的哪一行？
    ├────────────────────┤
    │   旧的 rbp 值        │  ← 调用者 main 的栈底指针
    ├────────────────────┤
    │   func 的局部变量    │  ← 包括你声明的 char buf[16]
    │   比如 buf[15]      │
    │        buf[14]      │
    │        ...          │
    │        buf[0]       │  ← 数组从低地址向高地址填充
    ├────────────────────┤
    │   还没用的空间       │  ← rsp（栈顶指针）指向这里
    └────────────────────┘
      低地址（地址数值小）
```

### 🔥 关键洞察：栈溢出的原理

```
buf 声明了 16 字节，数组填充方向是从低地址向高地址。

但是！buf 在"返回地址"的下方（地址更小）。

如果你往 buf 里写 100 个字节，多出来的 84 个字节会：
  1. 先覆盖 func 的其他局部变量
  2. 再覆盖"旧的 rbp"
  3. 最后覆盖 ★"返回地址"★

一旦返回地址被改了，func 返回时就会跳到攻击者指定的地址去执行代码！
```

### 💻 一个故意溢出的例子

```c
#include <stdio.h>
#include <string.h>

void vulnerable(char *input) {
    char buf[16];                // 只分配了16字节！
    strcpy(buf, input);          // 不检查长度，直接复制
    printf("buf内容: %s\n", buf);
}                                // 返回时从"返回地址"处取地址跳转

int main() {
    char long_input[100];
    memset(long_input, 'A', 99); // 99个'A'
    long_input[99] = '\0';
    vulnerable(long_input);      // 传入99字节 → buf只有16字节 → 溢出！
    return 0;
}
// 运行结果：段错误（Segmentation Fault）→ 返回地址被 'A' 覆盖了
```

### 堆的图解

```
      低地址
    ┌────────────┐
    │ 已申请块1   │ ← malloc 分配
    ├────────────┤
    │ 已申请块2   │
    ├────────────┤
    │ 空闲空间    │ ← 从这里分配新块
    ├────────────┤
    │  ...       │
    └────────────┘
      高地址

堆向高地址增长，栈向低地址增长——两者相向而行。
```

### 栈 vs 堆 速查表

| 特性 | 栈 Stack | 堆 Heap |
|------|---------|---------|
| 管理方式 | 自动（编译器管） | 手动（malloc/free） |
| 大小 | 小（通常 8MB） | 大（受物理内存限制） |
| 速度 | 快 | 较慢 |
| 增长方向 | 高→低 | 低→高 |
| 生命周期 | 函数返回即释放 | 手动 free 才释放 |
| 典型漏洞 | 栈溢出、ROP | 堆溢出、UAF、Double Free |

## 5. 📌 网安必知的几个危险函数

这些函数 **不检查目标缓冲区长度**，是绝大多数栈溢出的罪魁祸首：

### 💣 头号通缉犯

| 函数 | 为什么危险 | 生活类比 | 安全替代 |
|------|-----------|---------|---------|
| `gets(buf)` | 读到换行为止，不管buf多大 | 往杯子里倒水，不看杯子满了没 | `fgets(buf, n, stdin)` |
| `strcpy(dst, src)` | src多长就复制多长 | 大箱子往小箱子搬，硬塞 | `strncpy(dst, src, n)` |
| `strcat(dst, src)` | 往dst后面拼，不管够不够 | 往满杯子里续水 | `strncat(dst, src, n)` |
| `sprintf(buf, fmt)` | 格式化输出不管buf多大 | 无限长的小票打印机 | `snprintf(buf, n, fmt)` |
| `scanf("%s", buf)` | 读到空白为止，不管buf多大 | 同上 | `scanf("%15s", buf)` |

### 🛡️ 记住一个规律

```
安全的函数名字里都带 "n"：
  strncpy, strncat, snprintf, fgets

但 "n" 不是银弹——你必须正确设置 n 的值才有用。
```

## 6. GDB 调试——必须会的基本操作

### 启动调试

```bash
gcc -g -fno-stack-protector -o vuln vuln.c  # -g 加调试符号
gdb ./vuln
```

### GDB 核心命令速查

```
运行控制:
  r/run          — 运行程序
  b func         — 在函数 func 打断点
  b *0x400123    — 在指定地址打断点
  n/next         — 执行下一行（不进入函数内）
  s/step         — 执行下一行（会进入函数内）
  c/continue     — 继续运行到下一个断点

查看信息:
  p var          — 打印变量值        p/x var（十六进制）
  p &var         — 打印变量地址
  i r            — 查看所有寄存器( info registers )
  x/16xw $rsp    — 查看栈顶开始的16个字（十六进制）
  x/s addr       — 把该地址当字符串显示
  disas func     — 反汇编 func 函数
  bt             — 查看调用栈（backtrace）

退出:
  q/quit
```

### 🔧 用 GDB 观察栈溢出

```bash
(gdb) b vulnerable       # 在危险函数打断点
(gdb) r                  # 运行
(gdb) x/20xw $rsp        # 调用 strcpy 前看栈
(gdb) n                  # 执行 strcpy
(gdb) x/20xw $rsp        # 调用 strcpy 后看栈——返回地址变了！
```
