# 🏁 起点——从这里开始

## 怎么用这个知识库

这里每一篇都是可以直接打开的**学习材料**，不是模板，不是大纲。

**阅读顺序：** 数字越小越基础，建议按顺序读。但也可以直接跳到感兴趣的部分——每篇都尽量自包含，碰到不懂的名词翻回前面查。

**电脑上看：** 用 Typora / Obsidian / VS Code 打开，或者直接记事本看 Markdown 源码也行。
**手机上看：** 推送到 GitHub 后，手机浏览器打开 GitHub 仓库就能看，排版正常。

**核心规律：** 所有安全漏洞的本质，都是"程序做的事超出了设计者预期的范围"。理解了底层怎么跑的，你就知道为什么会有漏洞。

## ⚠️ 三条铁律

1. **先理解原理，再碰工具。** 不要一上来就开 Kali Linux 点菜单——那样你永远只停留在"会用工具"。
2. **每学一个漏洞，必须亲手复现。** 靶场链接在每篇末尾。
3. **用自己的话记。** 读完一篇后，在末尾空白处用一句话写下你的理解。能说清楚才是真的会了。
