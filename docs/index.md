# 🏁 起点——从这里开始

## 这个知识库是什么？

这是一个**从零开始**的网络空间安全自学教程。不是大纲，不是模板，不是"推荐文章列表"——每一篇都是可以直接阅读、可以动手操作的完整学习材料。

### 它适合谁？

| 你的情况 | 判断 |
|---------|------|
| 编程零基础，但愿意学 | ✅ 适合，01 章从"内存是什么"开始讲 |
| 写过代码但没接触过安全 | ✅ 最佳读者，你会惊叹"原来漏洞是这么来的" |
| 计算机专业学生，想走安全方向 | ✅ 帮你把零散知识串成体系 |
| 已经在做渗透但原理不透 | ✅ 跳到你薄弱的那章补基础 |
| 只想学"怎么黑别人网站" | ❌ 不适合，这里不教脚本小子的操作 |

### 它不教什么？

- ❌ 不教"下载 XX 工具点一下就能黑"——这里讲的是工具背后的原理
- ❌ 不教绕过杀毒软件的具体手法——这是攻防对抗的最后一环，不是第一环
- ❌ 不是 CVE 漏洞库的翻译——我们精选经典漏洞讲原理，不罗列编号
- ✅ 教的是：**程序怎么跑、数据怎么传、漏洞怎么产生、攻击怎么构造**

## 🔥 为什么网安必须从底层学起？

来看一个你每天都在用的场景：

### 你在超市买完东西，收银员亮出收款二维码，你打开微信扫码支付。

这 15 秒里发生了什么？

```
你的手机                          收银台                        微信服务器
                                                                
① 摄像头采集二维码图像                                             
    ↓                                                             
   图像处理算法解码二维码                                           
   提取: 商户ID + 订单号 + 金额                                     
                                                                
② 微信 App 建立 HTTPS 连接 ──────────────────────────────→ 微信服务器
   TLS 握手: 验证证书、交换密钥                                    
   (05-密码学: RSA密钥交换 + AES对称加密)                            
                                                                
③ 你输入支付密码 / 按指纹                                          
   手机安全芯片验证身份                                             
   密码哈希后传输 (05-密码学: SHA-256哈希)                          
                                                                
④                          → 后台处理 ←                            
                             查询余额 (06-Web安全: API安全)          
                             检查订单                               
                             冻结金额 (01-C语言: 内存中的数据结构)     
                             扣款+加款 (03-体系: CPU执行指令)         
                             记录流水                               
                                                                
⑤                          ← 返回结果 ←                            
   { "status": "success",                                           
     "amount": "32.50" }    支付成功！                              
   (04-网络: HTTP响应)      (04-网络: TCP确认)                      
```

**这 15 秒涉及的知识点：** 二维码编码、图像处理、WiFi/4G通信、TCP/IP协议栈、TLS握手、RSA密钥交换、AES加密、SHA-256哈希、数字证书验证、HTTP请求、API安全、SQL查询、内存管理、并发处理、支付回调签名验证……

**每一个环节都可能出安全漏洞。** 网络安全不是一门独立学科——它是计算机科学所有分支在"对抗"这个维度上的交汇点。

你现在打开这个知识库，就是要**一层层拆开这个"15秒"**，搞懂每一步是怎么实现的、每一步可能在哪个环节被攻击。

## 📱 怎么用这个知识库

**电脑上看（推荐）：** Typora / Obsidian / VS Code 打开 Markdown 文件，代码高亮、表格渲染都很好。

**手机上看：** 浏览器打开 `https://king-sw-0106.github.io/security-kb/`，Material 主题做了移动端适配。

**怎么学最高效：**
1. **先看"门前检查"。** 每章开头有3-5个自检问题。答不上来？回去翻前一章。
2. **代码必敲。** 看十遍不如手敲一遍。每个代码块都是可运行的。
3. **做完"小实验"。** 每节末尾有改参数看效果的实验，花5分钟试试。
4. **用自己的话记。** 读完一节后，在末尾空白处写一句总结。能讲清楚才是真会了。
5. **用好书签系统。** 00-目录与进度里有书签栏，记下总记不住的地方，回头反复看。

## ⚠️ 开始前必须知道的

1. **你学的是"防御式安全"。** 理解漏洞是为了写出更安全的代码、设计更安全的系统。不要拿学到的技术去做违法的事——破坏永远比建设容易，但建设才是你值钱的地方。

2. **你一定会卡住。** 07-二进制安全那章，几乎每个人第一次都看不懂。这是正常的。回到 01 和 03 重新看栈帧那部分，再回来看——你会发现"哦原来是这样"。

3. **不要跳章。** 最大的误区就是：觉得 Web 安全有意思，跳过计算机体系直接学 SQL 注入。你会学会怎么用 sqlmap，但你永远不知道为什么参数化查询能防注入。前者叫脚本小子，后者叫安全工程师。

4. **动手 > 阅读。** 这个知识库读一遍大约 50 小时。但如果你每读一节就动手做一遍实验，可能需要 100 小时。**后者的效果是前者的 10 倍。**
